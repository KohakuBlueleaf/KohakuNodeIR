# examples

Example programs and workflows demonstrating KIR and KohakuNodeIR.

## Directories

### kir_basics/

Ten basic `.kir` programs with matching Python runner scripts. Covers the core
language features:

| File | Topic |
|---|---|
| `hello_world.kir` | Minimal program |
| `basic_math.kir` | Arithmetic operations |
| `string_processing.kir` | String manipulation |
| `data_pipeline.kir` | Chained data processing |
| `branching.kir` | Conditional branching |
| `control_flow.kir` | Labels, jumps, branches |
| `switch_demo.kir` | Switch/case dispatch |
| `parallel_demo.kir` | Parallel execution blocks |
| `subgraph_demo.kir` | Subgraph definitions and calls |
| `mixed_mode.kir` | Combining sequential and dataflow modes |

Each `.py` file registers the required nodes and runs the corresponding `.kir`
program. Run all of them with:

```bash
python examples/kir_basics/run_all.py
```

### kirgraph_pipeline/

Full L1 -> L2 -> L3 compilation pipeline demo. Shows:

- `source.kirgraph` -- L1 node graph definition
- `compiled_l2.kir` -- compiled L2 output (with `@meta` annotations)
- `compiled_l3.kir` -- compiled L3 output (stripped, executable)
- `decompiled.kirgraph` -- round-trip decompilation back to L1
- `demo.py` -- Python script that runs the full pipeline

### editor/

Example files generated by or used with the kir-editor. Includes KirGraph JSON
files (`.kirgraph`) and their corresponding KIR text (`.kir`), plus round-trip
test files (`rt_*.kir`).

### custom_backend/

Custom execution backend examples. Shows how to subclass `ExecutionBackend` to
intercept and customize function dispatch:

| File | Topic |
|---|---|
| `logging_backend.py` | Backend that logs every call with timing |
| `caching_demo.py` | `CachingBackend` with call counting to prove caching works |
| `stateful_backend.py` | Backend that gives nodes persistent state across runs |

```bash
python examples/custom_backend/logging_backend.py
python examples/custom_backend/caching_demo.py
python examples/custom_backend/stateful_backend.py
```

### comfyui_convert/

ComfyUI workflow conversion examples. Converts ComfyUI JSON workflow files into
KirGraph and KIR formats:

- `example_workflow.json` / `example_workflow2.json` -- source ComfyUI workflows
- `convert.py` -- conversion script
- `converted.kirgraph` / `converted_l2.kir` / `converted_l3.kir` -- outputs

## Running

Most examples require the `kohakunode` package to be installed:

```bash
pip install -e .     # from repo root
python examples/kir_basics/run_all.py
python examples/kirgraph_pipeline/demo.py
```
