# kir-editor frontend

Vue 3 + Vite single-page application for the KIR visual editor.

## Key dependencies

| Package | Purpose |
|---|---|
| Vue 3 | UI framework |
| Pinia | State management (graph store, editor state) |
| Monaco Editor | Code view (`.kir` text editing) |
| Element Plus | UI components (panels, buttons, dialogs) |
| Vue Router | View navigation |
| UnoCSS | Utility-first CSS (Attributify + Icons presets) |
| Axios | HTTP client for backend API calls |

## WASM integration

The frontend loads `kohakunode_rs.wasm` (built from `src/kohakunode-rs/`) for
in-browser parsing and layout. The WASM module lives in `src/wasm/` and provides:

- `parse_kir()` -- parse `.kir` source to AST
- `kir_to_graph()` / `compile_kirgraph()` -- convert between KirGraph and AST
- `auto_layout()` / `optimize_layout()` -- graph layout algorithms
- `write_kir()` -- serialize AST back to `.kir` text

No backend is needed for editing -- only for execution.

## Source structure

```
src/
  api/           API client (Axios calls to backend)
  compiler/      Frontend compilation helpers
  components/    Vue components (node graph, blocks, panels)
  composables/   Vue composables (shared reactive logic)
  editor/        Monaco editor integration
  layout/        Layout utilities bridging WASM
  parser/        KirGraph loader and parser wrappers
  stores/        Pinia stores (graph state, editor state)
  styles/        Global CSS
  utils/         Shared utilities
  wasm/          WASM module (generated by build-wasm.sh)
  App.vue        Root component
  main.js        App entry point
```

## Development

```bash
npm install          # install dependencies
npm run dev          # start Vite dev server (http://localhost:5173)
npm run build        # production build to dist/
npm run preview      # preview production build
npm run build:wasm   # rebuild WASM module from Rust source
```

## Testing

```bash
npm test             # run tests once (vitest)
npm run test:watch   # watch mode
```

Test runner: Vitest with happy-dom for DOM simulation and @vue/test-utils.
